Analyzed GitHub Pages visibility requirements ja käytännön vaikutukset

Ilmaisella GitHub-tilillä repon pitää olla julkinen — Pages toimii ilmaistilillä vain julkisilla repoilla. Yksityisistä repoista Pagesin saa julkaistua vain maksullisilla tileillä (Pro, Team, Enterprise). GitHub

Mutta tähän liittyy yksi olennainen juttu, joka kannattaa tietää: vaikka olisit maksullisella tilillä ja pitäisit repon yksityisenä, itse sivusto on silti julkinen kaikille, joilla on osoite. Ja koska tämä on staattinen sivu (pelkkä HTML/CSS/JS), kuka tahansa näkee koko koodin selaimen "näytä lähdekoodi" -toiminnolla. Yksityinen repo piilottaa siis vain repon näkymän GitHubissa, ei julkaistun sivun koodia. GitHub

Käytännössä tälle demolle: laita repo vain julkiseksi. Tiedostossa ei ole oikeita avaimia, salasanoja eikä autentikointia — kaikki on esimerkkidataa — joten mitään salattavaa ei ole. Yksityinen repo ei tässä tapauksessa toisi mitään hyötyä, koska front-end-koodi näkyy kävijöille joka tapauksessa.

Jos joskus myöhemmin haluat piilottaa oikeaa logiikkaa tai avaimia, ne eivät kuulukaan staattiselle sivulle collenkaan vaan backendiin (kuten sinulla jo on Supabasessa) — mutta se on eri juttu kuin tämä esittelysivu.

Moro! Tuo "none" johtuu yksinkertaisesti siitä, että repo on vielä tyhjä. GitHub sanoo sen itse asiassa suoraan tuossa: "You must first add content to your repository before you can publish a GitHub Pages site." Heti kun lataat index.html:n repoon, main-haara ilmestyy tuohon pudotusvalikkoon. Eli ladataan tiedosto ensin, sitten Pages-asetus aukeaa.

1. Lataa index.html repoon (helpoin tapa, selaimessa):

  1. Lataa tämän chatin yltä jakamani index.html koneellesi (jos et jo tehnyt).
  2. Mene repon Code-välilehdelle: github.com/elias1998-fi/TotalParkingweb
  3. Tyhjässä repossa näkyy linkki "uploading an existing file" — klikkaa sitä. (Tai jos repossa on jo esim. README, paina Add file → Upload files.)
  4. Raahaa index.html latausalueelle tai valitse se selaimella.
  5. Vieritä alas ja paina vihreää Commit changes -nappia.

Nyt tiedosto on main-haarassa.

2. Kytke Pages päälle:

  1. Mene takaisin Settings → Pages.
  2. Source: Deploy from a branch (on jo valittuna).
  3. Branch: nyt valikossa lukee main "none":n sijaan → valitse se.
  4. Kansio: / (root).
  5. Paina Save.

Odota ~1–2 minuuttia ja päivitä sivu. Ylös ilmestyy vihreä palkki: "Your site is live at…". Osoite on:

https://elias1998-fi.github.io/TotalParkingweb/

Myöhemmin kun haluat päivittää sivun: avaa index.html repossa, paina kynäkuvaketta (Edit) ja liitä uusi sisältö → Commit. Tai lataa Add file → Upload files -kautta saman niminen tiedosto uudelleen, niin se korvaa vanhan. Pages rakentaa sivun automaattisesti uusiksi parin minuutin sisällä jokaisen committin jälkeen.

Huom: tuo Visibility-kohdan "GitHub Enterprise" ja "Start free for 30 days" on vain maksullisen tilin yksityisjulkaisua varten — älä koske siihen. Sinun reittisi on tuo ilmainen julkinen Pages, joka tulee tiedoston lataamisen jälkeen.
